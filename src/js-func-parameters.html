    • 1. Introduction to Function Parameters:

    Explanation (On Chalkboard): Function parameters are placeholders that allow you to pass values into a function when you call it. Think of them as variables that are defined when you declare the function and are later filled in with specific values when you use the function.
    Code Example (In Code Editor): // Function with parameters function greet(name) { console.log("Hello, " + name + "!"); } // Calling the function with an argument greet("Alice"); // Output: Hello, Alice!

    2. Defining and Using Parameters:

    Explanation (On Chalkboard): Parameters are declared inside the parentheses of the function declaration. When you call the function, you provide actual values, called arguments, which are assigned to the parameters.
    Code Example (In Code Editor): // Function with multiple parameters function add(a, b) { return a + b; } // Calling the function with arguments let sum = add(5, 3); // sum will be 8

    3. Default Parameters:

    Explanation (On Chalkboard): Default parameters allow you to provide default values for parameters. If an argument is not provided when the function is called, the default value is used.
    Code Example (In Code Editor): // Function with default parameter function power(base, exponent = 2) { return Math.pow(base, exponent); } let result1 = power(3); // Uses exponent 2 by default let result2 = power(3, 4); // Uses exponent 4

    4. Rest Parameters:

    Explanation (On Chalkboard): Rest parameters gather remaining arguments into an array. This is useful when you want to pass a variable number of arguments to a function.
    Code Example (In Code Editor): // Function with rest parameter function sumAll(...numbers) { return numbers.reduce((total, num) => total + num, 0); } let totalSum = sumAll(1, 2, 3, 4, 5); // Result: 15

    5. Named Parameters (Object Destructuring):

    Explanation (On Chalkboard): You can pass parameters as an object and destructure them within the function. This provides clarity when calling functions with multiple arguments.
    Code Example (In Code Editor): // Function with named parameters using object destructuring function displayPersonInfo({ name, age }) { console.log(`Name: ${name}, Age: ${age}`); } let person = { name: "John", age: 30 }; displayPersonInfo(person); // Output: Name: John, Age: 30

    6. Passing Functions as Parameters:

    Explanation (On Chalkboard): Functions can also be passed as parameters. This is commonly used for callbacks in higher-order functions, allowing for dynamic behavior.
    Code Example (In Code Editor): // Function that uses a callback function function processArray(arr, callback) { let result = []; for (let item of arr) { result.push(callback(item)); } return result; } let numbers = [1, 2, 3]; let squaredNumbers = processArray(numbers, num => num * num); // [1, 4, 9]

    7. Understanding Scope and Parameters:

    Explanation (On Chalkboard): Parameters have their own scope within the function, separate from the rest of the code. This helps prevent naming conflicts with variables outside the function.
    Code Example (In Code Editor): let globalVar = "I'm global"; function exampleScope(localVar) { console.log(globalVar); // Accessible console.log(localVar); // Accessible } exampleScope("I'm local"); console.log(globalVar); // Accessible console.log(localVar); // Error: localVar is not defined
